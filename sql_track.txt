A scalar subquery is a subquery that returns:
✔ Exactly 1 row AND 1 column (a single value)
eg syntax: SELECT salary FROM (subquery) AS t;
📘 SQL Subquery & Alias Notes (Unit-5)
🔹 Scalar Subquery
A scalar subquery returns exactly 1 row and 1 column
It produces a single value

Example:

SELECT (SELECT MAX(salary) FROM Employee);
Used in SELECT, WHERE, or RETURN statements
🔹 Derived Table (Subquery with alias)
A subquery used in FROM clause is called a derived table
It must have an alias using AS

Example:

SELECT salary
FROM (SELECT salary FROM Employee) AS t;
🔹 Role of alias (AS t)
AS t is just a temporary table name
It does NOT affect scalar property
It helps SQL refer to subquery result as a table
🔹 Column names in derived table
By default, derived table keeps same column names as inner query

Example:

SELECT salary
FROM (SELECT salary FROM Employee) AS t;

→ column name remains salary

If renamed inside subquery:

SELECT salary AS s
FROM Employee

→ derived table column becomes s

# Write your MySQL query statement below
select score, DENSE_RANK(order by Scores) as rank
from  order by Scores desc;
 what is the error and  wrong syntax:
 ❌ Missing table name after FROM
 ❌ Wrong syntax of DENSE_RANK()
 ❌ ORDER BY inside function is incorrect
 ❌ Column naming mismatch (score vs Scores)
 ✅ Correct Query (LeetCode 178)
SELECT score,
       DENSE_RANK() OVER (ORDER BY score DESC) AS rank
FROM Scores;
🧠 How this works (very important)
Step 1: FROM Scores

Take the full table.

Example:

+-------+
| score |
+-------+
| 100   |
| 90    |
| 90    |
| 80    |
+-------+
Step 2: ORDER BY score DESC inside window function

We sort scores internally (not output order yet):

100 → highest
90
90
80
Step 3: DENSE_RANK()

Assign ranks:

100 → rank 1
90 → rank 2
90 → rank 2 (same rank, no gap)
80 → rank 3
✅ Final Output
+-------+------+
| score | rank |
+-------+------+
| 100   | 1    |
| 90    | 2    |
| 90    | 2    |
| 80    | 3    |
+-------+------+
🔥 Key Concept (Important for interviews)
DENSE_RANK() → no gaps in ranking
RANK() → gaps appear
ROW_NUMBER() → always unique numbers
